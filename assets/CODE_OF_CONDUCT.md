# Code of Conduct

## LocalPulse Payment Gateway™ — Professional Standards

This Code of Conduct governs participation in the LocalPulse Payment Gateway™ repository, including code contributions, code reviews, issue discussions, and all related communications. This applies to LocalPulse employees, approved contractors, and technology partners.

---

## Our Standards

The LocalPulse Payment Gateway™ processes financial transactions for thousands of businesses. The seriousness of this responsibility demands the highest standards of professional conduct from everyone who works on it.

### Expected Behavior

All contributors are expected to:

- Act professionally and treat all collaborators with respect
- Provide constructive, specific, and actionable feedback in code reviews
- Accept feedback graciously and engage with it on its technical merits
- Prioritize security and customer safety over shipping velocity
- Escalate concerns about security or compliance through proper channels
- Protect the confidentiality of proprietary code, customer data, and internal systems
- Honor NDAs and all contractual obligations with LocalPulse.pro™
- Represent LocalPulse.pro™ professionally in all external communications
- Report potential security or compliance issues immediately

### Prohibited Behavior

The following are explicitly prohibited:

- Harassment, discrimination, or hostile behavior toward any collaborator
- Sharing or exposing proprietary code, internal documentation, or customer data
- Introducing intentional vulnerabilities, backdoors, or malicious code
- Attempting to access systems or data beyond your authorized scope
- Misrepresenting the security posture of any code you review or submit
- Pressuring others to approve PRs or bypass security reviews
- Making unauthorized public statements about security vulnerabilities
- Retaliating against anyone who raises a legitimate security or compliance concern

---

## Reporting Concerns

### Professional Conduct Issues

If you experience or witness behavior that violates this Code of Conduct:

1. **Direct resolution** — If safe and appropriate, address it directly with the individual
2. **Manager escalation** — Report to your manager or the LocalPulse engineering lead
3. **HR escalation** — For LocalPulse employees: contact HR at hr@localpulse.pro
4. **Contractor escalation** — Contact your engagement manager or engineering@localpulse.pro

All reports will be handled confidentially. We do not tolerate retaliation against anyone who raises a good-faith concern.

### Security or Compliance Concerns

If you observe a potential security vulnerability, compliance gap, or suspicious code change:

1. **Do not discuss publicly** — Do not post in Slack, GitHub comments, or any public channel
2. **Report immediately** — Email security@localpulse.pro with details
3. **Preserve evidence** — Do not modify or delete anything related to the concern
4. **Follow instructions** — Await guidance from the security team before taking further action

Given our PCI DSS obligations, failure to report a known security issue is itself a serious violation.

---

## Enforcement

### Review Process

All reported violations are reviewed by the LocalPulse security or HR team (depending on the nature of the issue) within 5 business days.

### Consequences

Violations may result in, depending on severity:

| Severity | Example | Consequence |
|----------|---------|-------------|
| Minor | Unprofessional comment in code review | Written warning; coaching |
| Moderate | Repeated disrespectful behavior | Removal from repository; contract review |
| Serious | Unauthorized data access; intentional backdoor | Immediate access revocation; legal action |
| Critical | Data breach; deliberate sabotage | Termination; law enforcement referral |

LocalPulse.pro™ reserves the right to revoke repository access, terminate contracts, and pursue legal remedies for violations involving proprietary code, customer data, or deliberate security sabotage.
